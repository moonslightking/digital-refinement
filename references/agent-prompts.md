# 7-Agent 并行采集：详细提取指令

Phase 1 执行时阅读此文件。每个 Agent 独立运行，输出可追溯、可比较的认知证据。

## 通用要求

每个 Agent 的输出都必须包含：

- 原始材料引用或可访问来源。
- 日期，无法精确到日时至少精确到月或年。
- 目标人物当时的受众、场合和表达目的。
- 与认知结构相关的观察，而不是泛泛摘要。
- 对证据质量的初步判断：`authenticity`、`representativeness`、`explanatory_power`、`verifiability`。

避免三类错误：

- 只摘录结论，不提取推理路径。
- 把后见之明写成对象当时的认知。
- 只收集支持性材料，忽略失败、矛盾和批评。

## Agent 1：公开著述分析

### 任务

从目标人物的一手著作、文章、演讲稿中提取认知证据。

### 提取要素

1. **核心论点**：此文的中心主张是什么？
2. **论证结构**：他用什么逻辑推导出结论？演绎、归纳、类比、反证，还是经验归纳？
3. **关键概念**：他引入或反复强调哪些概念？这些概念是原创、借用，还是重新定义？
4. **重复模式**：是否与其他著述中的思维框架重复出现？
5. **原话摘录**：保留能展现思维方式的原始表述。
6. **确定性信号**：他如何表达确定、不确定和承认无知？

### 质量评分指引

- Authenticity：一手原文 → H；编辑整理版 → M；转述 → L。
- Representativeness：核心主题反复出现 → H；偶尔提及 → M；仅一次 → L。
- Explanatory Power：完整论证链 → H；有理由但不完整 → M；仅有结论 → L。
- Verifiability：有原始出版物可查 → H；仅有转录 → M；来源不明 → L。

### 材料溯源校验

Agent 1 输出后必须执行：

1. **出处链标注**：每条材料标注原始发布平台 → 转载路径。
2. **署名检查**：对署名不明确或可疑的材料执行风格一致性检查。
3. **降级处理**：风格不一致或署名不明确的材料，`authenticity` 直接标 L，不用于心智模型提取。
4. **标注格式**：

```json
{
  "provenance": "雪球原创帖 → 博客转载",
  "authorship_verified": true,
  "style_check_notes": "与确认材料风格一致"
}
```

## Agent 2：对话记录分析

### 任务

从访谈、问答环节、对话记录中提取即时思维。对话常比著述更能显示反应方式、思考深度和回避点。

### 特别关注

1. 即兴回答的逻辑结构。
2. 被追问时是深化论证、承认不知道，还是转移话题。
3. 如何处理超出认知范围的问题。
4. 系统性回避的话题。
5. 幽默、隐喻和类比的使用。
6. 是否根据提问者背景调整回答层级。

### 注意事项

公开问答仍有表演性。私人对话通常更接近真实认知状态，但必须确认来源可靠性和传播路径。

## Agent 3：行为证据分析

### 任务

从实际投资决策、资源配置、时间分配等行为中推断认知模式。行为证据用于校验言论，而不是简单说明成败。

### 提取要素

1. **决策内容**：做了什么？
2. **决策时间**：最好精确到周，至少精确到月。
3. **当时的信息环境**：能看到什么，不能看到什么。
4. **替代方案**：当时有哪些其他选择，为什么排除。
5. **时间压力**：深思熟虑还是快速反应。
6. **后续行为**：是否调整，持有或执行多久。
7. **决策结果**：结果如何，但避免结果偏差。

### 特别关注

- **言行不一致**：说法与行为不一致时，记录为重要认知证据。
- **持仓集中度**：资金分配反映风险偏好和确定性要求。
- **不做的决策**：选择不投、不卖、不回应，也可能揭示边界。

## Agent 4：他人评价分析

### 任务

收集同行、合作者、批评者对目标人物的评价，作为交叉验证源。

### 提取要素

1. 评价者身份和与目标人物的关系。
2. 关于目标人物思维方式的具体描述。
3. 评价者是否有足够接触来做判断。
4. 多个独立评价是否指向相同特征。

### 注意事项

- 合作者可能美化，竞争对手可能贬低。
- 公众评价常基于刻板印象。
- 必须标注评价对应的时期。

## Agent 5：智识谱系分析

### 任务

追踪目标人物的知识来源和思想影响链。不在本阶段评价影响的对错。

### 提取要素

1. 他公开引用了谁的思想，频率如何。
2. 未明确引用但结构明显相似的前人或同代人。
3. 他在哪些方面比影响者走得更远。
4. 他在哪些方面仍受影响者框架限制。
5. 哪些思想是他原创或再组合的。
6. 他明确拒绝了谁的思想，以及拒绝理由。

### 投资者常见谱系

- 学术导师或早期职业导师。
- 投资哲学源头人物，如格雷厄姆、费雪、凯恩斯、芒格。
- 跨学科影响，如心理学家、科学家、哲学家。
- 同代人的交叉影响。

## Agent 6：反面证据分析

### 任务

专门收集与前五个 Agent 提取结果相矛盾或不利的证据。

### 三类搜索方向

**a. 他人的批评和质疑**
- 理性的批评者如何反驳他的方法论？
- 同行中谁对他的判断持反对意见？
- 批评是否有实质性论据？

**b. 对象事后承认的失误**
- 他公开承认过哪些投资失误？
- 失误根因是认知偏差、信息不足、执行问题，还是环境变化？
- 他是否从失误中形成新的判断方式？

**c. 判断被市场证伪的案例**
- 哪些公开预测被后续走势证伪？
- 证伪后他是承认错误、沉默，还是重新解释？
- 自我叙述是否有美化或简化历史的倾向？

### 补充搜索

- 认知盲区：有没有系统性忽略的重要因素？
- 幸存者偏差：成功中有多少来自运气或时代背景？

## Agent 7：个人环境快照

### 任务

为每个重要时间节点记录目标人物的个人环境。宏观和市场背景由 MCB 覆盖，Agent 7 只补充人物自身状态。

### 个人环境维度

1. **资产/仓位状态**：资产管理规模、仓位水平、持仓结构。
2. **近期交易结果**：重大盈亏事件及可能心理影响。
3. **生活事件**：健康、家庭、搬迁等可能影响判断的变化。
4. **社交圈变化**：与哪些人交流频繁，信息源是否变化。
5. **信息获取方式**：研究习惯、渠道、调研方式。
6. **市场互动方式**：公开写作、社群交流、公开持仓或沉默。

### 输出格式

```json
{
  "date": "YYYY-MM",
  "mcb_ref": "CN-YYYY-MM",
  "personal_environment": {
    "aum_status": "管理约X亿，满仓",
    "recent_pnl": "重仓股近月涨幅30%，心态偏乐观",
    "life_events": "无重大变化",
    "social_circle": "与XX、YY交流频繁",
    "info_channels": "主要通过实地调研和行业报告",
    "market_engagement": "每周发布2-3篇分析帖"
  }
}
```

### 为什么个人环境不可省略

同样的市场环境下，满仓和空仓的人心态不同；刚经历大亏和刚经历大赚的人判断方式也不同。个人状态是生态理性的一部分。
