# 语境标注操作手册

Phase 2 执行时阅读此文件。语境标注的目标是让每条认知证据都带有产生时的环境信息，使后续使用者能判断它的适用范围和失效条件。

## 目的

语境标注解决三个问题：

1. 避免把特定市场环境下的判断误当成通用原则。
2. 避免把公开表达、私人表达、事后反思混在一起。
3. 让生成的人格在新问题上能提示环境相似性和不确定性。

理论基础：

- **历史语境主义**：文本是特定语境中的言语行为，不能脱离语境理解。
- **生态理性**：认知方式与环境互相匹配，判断好坏取决于当时可见信息和约束条件。

## 标注结构

```json
{
  "statement_id": "S001",
  "statement": "原始内容",
  "date": "YYYY-MM-DD",
  "mcb_ref": "CN-2015-06",
  "personal_context": "刚从实地调研回来，对创业板科技股信心很强",
  "audience_and_purpose": "雪球公开帖，面向散户粉丝，分享看法",
  "context_dependency": "high",
  "dependency_evidence": "Step1=是 → Step3=是 → Step4=当创业板估值回归合理时失效"
}
```

### 字段说明

- **mcb_ref**：引用 MCB 月度切片 ID，格式为 `{market}-{YYYY}-{MM}`。单月用字符串，横跨多月用字符串数组。
- **personal_context**：人物个人体感和状态，不写宏观环境，控制在 50 字以内。
- **audience_and_purpose**：对谁说的、为什么说。
- **context_dependency**：`high | medium | low`。
- **dependency_evidence**：记录判定路径和失效条件。

## 标注流程

### Step 1：选择需要标注的数据

优先标注：

- 所有含有明确立场或判断的言论。
- 所有重大决策。
- 所有与其他时期存在矛盾的内容。
- Phase 4 中会被引用为证据的内容。

可以跳过：

- 纯粹知识陈述。
- 已被大量引用且语境明确的名言，但仍需记录出处。

### Step 2：填写语境字段

**personal_context 示例**

- "基金当年回撤约15%，已提前减仓"
- "刚完成对XX公司的实地调研，印象极佳"
- "满仓状态，近一个月收益+20%，心态偏乐观"
- "刚经历重仓股暴跌30%，情绪低落期"

**audience_and_purpose 示例**

- "雪球公开帖，面向散户粉丝，分享看法"
- "博客年度总结，面向长期读者，年度反思"
- "私人社群发言，面向同行基金经理，即兴讨论"
- "年度投资者大会，面向LP，解释当前持仓策略"

### Step 3：判定语境依赖度

#### high 依赖

判断强依赖当时的市场、估值、政策、流动性或个人状态。

示例：

- "现在是买入中国股票的好时机"
- "现金为王"
- "我不会碰加密货币"

#### medium 依赖

原则相对稳定，但应用方式随环境变化。

示例：

- "我偏好集中投资而非分散"
- "能力圈很重要"

#### low 依赖

接近通用方法论，语境改变不影响核心含义。

示例：

- "反过来想，总是反过来想"
- "投资中最重要的是避免做蠢事"

### Step 4：补充失效条件

对 `high` 依赖内容，在 `dependency_evidence` 中记录完整判定路径和失效条件：

```json
{
  "statement": "现在是买入中国股票的好时机",
  "context_dependency": "high",
  "dependency_evidence": "Step1=是（含'现在'时间指代）→ Step3=是（结论依赖当时估值处于历史低位）→ Step4=失效条件：当中国市场估值不再处于历史低位时"
}
```

## 与 MCB 的交互规则

### 如何查找 MCB 切片

1. 根据言论或决策日期确定月份。
2. 在 `MCB_CN.jsonl` 或 `MCB_US.jsonl` 中查找对应 ID，例如 `CN-2015-06`。
3. 如果目标月份标记为数据不完整，在 `dependency_evidence` 中注明薄弱层级。

### MCB 数据不足时

- 不要自行补写自由文本替代宏观环境。
- 在标注中注明缺失层级。
- 如果缺失的是关键层级，在 `dependency_evidence` 说明对依赖度判定的影响。

## 常见陷阱

### 陷阱 1：过度标注

不要对每条数据都写长段背景。对 `low` 依赖内容，简短记录 `mcb_ref` 和个人状态即可。

### 陷阱 2：事后语境

不要用现在已知的结果来描述对象当时的环境。语境应反映对象当时能看到的世界。

### 陷阱 3：遗漏受众维度

同一个人对散户、机构、朋友、自己写日记时会采用不同表达层级。受众改变可能导致表达改变。

### 陷阱 4：忽略沉默

特定场合选择不讨论某个话题，本身也可能是重要语境信息。

### 陷阱 5：personal_context 混入宏观信息

`personal_context` 只写人物个人状态，不写宏观环境。宏观环境由 `mcb_ref` 覆盖。

错误：

```json
{
  "personal_context": "A股大涨，市场情绪火爆，个人满仓"
}
```

正确：

```json
{
  "personal_context": "满仓状态，近期收益良好"
}
```
