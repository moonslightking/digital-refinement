# 数字炼化：投资者认知蒸馏方法论框架

数字炼化是一套将投资者、交易员或投资 KOL 的认知结构从历史材料中提取出来，并转化为可运行认知人格的方法。它关注的不是传记事实，也不是观点摘要，而是一个人如何观察世界、如何形成判断、如何在不确定环境中做决策。

## 1. 适用范围

本方法适用于具备以下特征的目标人物：

- 有足够一手材料，如文章、演讲、访谈、股东信、社交媒体长帖。
- 有可观察决策或可验证判断，尤其是投资、交易、资产配置、企业经营相关决策。
- 活跃期足够长，可以观察认知演化。
- 目标不是神化人物，而是提取其稳定认知结构和典型偏差。

不适合炼化的对象：

- 只有二手评价，缺少直接表达。
- 决策不可观察，无法把言论与行为对齐。
- 材料时间跨度太短，只能得到静态画像。
- 用户只想要传记、金句、观点汇编，而不是认知人格。

## 2. 方法论前提

### 2.1 认知可结构化

一个人反复出现的判断方式、价值排序、概念偏好和风险处理方式，可以被抽象成心智模型、决策启发式、表达 DNA、认知偏误和诚实边界。

### 2.2 认知必须带语境

任何言论都发生在特定时间、受众、市场环境和个人处境中。脱离语境提取人格，会把阶段性判断误当成永恒原则。

### 2.3 认知在时间中演化

目标人物的认知不是静止的。炼化必须识别扩展、修正、收缩、分裂等变化，并说明哪些变化足以构成阶段分界。

### 2.4 行为证据优先于自我叙述

自我叙述有表演性和事后合理化风险。言论必须与实际行为、他人观察、反面证据、失败案例互相校验。

## 3. 核心概念

### 3.1 Cognitive Persona

认知人格不是角色扮演文本，而是一个可运行的推理接口。它应该回答：

- 这个人会优先看什么变量？
- 他如何权衡风险、赔率、时间和确定性？
- 他会在哪些情境下改变判断？
- 他有哪些盲区、偏误和诚实边界？

### 3.2 Macro Context Base

MCB 是宏观语境基座，用月度切片记录市场和宏观环境。它让不同炼化项目使用同一套时代坐标，避免每次手写宏观背景。

MCB 最少包含六层信息：

1. 宏观经济层
2. 股市层
3. 行业周期层
4. 政策事件层
5. 市场情绪层
6. 叙事层

### 3.3 Inheritable Layer

继承层是认知人格默认加载的结构，包含可以迁移到新问题上的认知运行方式：

- 心智模型
- 决策启发式
- 表达 DNA
- 认知偏误画像
- 价值排序与张力
- 智识谱系

### 3.4 Meta Layer

元信息层记录目标人物在特定社会角色下形成的自我规训、公开表达边界和历史行为习惯。它只用于历史语境，不应变成 agent 的硬性执行禁令。

### 3.5 Holdout Set

留出集是验证材料。它必须在正式提取前切分并隔离，避免炼化过程提前看见答案。

## 4. 总体流程

```text
Phase 0 → Phase 0.1 → Phase 0.5 → Phase 1 → Phase 2 → Phase 3 → Phase 4 → Phase 5 → Phase 6
可行性    MCB校验     留出集切分   数据采集   语境标注   演化建模   结构提炼   人格构建   质量验证
评估
```

## 5. Phase 0：可行性评估

### 目标

判断目标人物是否值得炼化，以及最终结果可能达到什么置信层级。

### 五维评分

| 维度 | 关注问题 | 低分处理 |
|------|----------|----------|
| 一手材料丰富度 | 是否有足够直接言论和原始文本？ | 低于 2 分则终止 |
| 决策可观测性 | 是否有公开决策和结果记录？ | 低于 2 分则终止 |
| 认知演化可追溯性 | 是否有足够长的时间跨度？ | 继续但降档 |
| 语境可还原性 | 是否能还原发表场合、受众、市场环境？ | 继续但标注缺口 |
| 现有分析深度 | 是否有外部研究可交叉验证？ | 继续但标注缺口 |

### 输出

`phase-0-assessment.md`，包括来源清单、评分、可行性结论、预期置信等级和主要数据缺口。

## 6. Phase 0.1：MCB 校验

### 目标

确认 MCB 已覆盖目标人物活跃期，避免 Phase 2 语境标注时临时补背景。

### 检查项

- 目标人物活跃年份是否被 MCB 覆盖。
- 关键月份是否有完整的六层数据。
- 中美市场是否分别有对应 JSONL 文件。
- 薄弱月份是否会影响语境依赖度判定。

### 输出

在 `meta.json` 中记录 MCB 覆盖范围、缺口月份和处理方式。

## 7. Phase 0.5：留出集切分

### 目标

把一部分材料从提取流程中隔离出来，供最终验证使用。

### 原则

- 先切分，再采集。
- manifest 只记录 ID、排除指令和 hash，不记录内容摘要。
- Phase 1-5 的任何 agent 不得读取留出集正文。
- Phase 6 解封时校验 SHA256。

### 推荐抽样

- 常规项目：总材料的 10%-15%。
- 材料有限项目：不少于 5 条。
- 留出集应覆盖不同时间段、材料类型和关键争议点。

## 8. Phase 1：多维数据采集

### 目标

从不同证据源并行提取认知证据，避免只靠公开著述形成单一画像。

### Agent 分工

| Agent | 任务 | 重点 |
|-------|------|------|
| Agent 1 | 公开著述 | 核心观点、论证结构、概念系统、确定性表达 |
| Agent 2 | 对话记录 | 即兴推理、追问反应、承认不知道的方式 |
| Agent 3 | 行为证据 | 实际投资、资源配置、持仓、交易后续 |
| Agent 4 | 他人评价 | 同行、合作者、批评者的观察 |
| Agent 5 | 智识谱系 | 信息源、导师、影响链、原创贡献 |
| Agent 6 | 反面证据 | 批评、失误、预测失败、选择性记忆 |
| Agent 7 | 个人环境 | 仓位、资产规模、生活事件、社交圈、信息渠道 |

### 质量评分

每条材料至少标注四个维度：

- `authenticity`：真实性
- `representativeness`：代表性
- `explanatory_power`：解释力
- `verifiability`：可验证性

## 9. Phase 2：语境标注

### 目标

为关键言论和决策补充可复用语境，使后续人格能判断何时适用、何时失效。

### 标注结构

```json
{
  "statement_id": "S001",
  "statement": "原始内容",
  "date": "YYYY-MM-DD",
  "mcb_ref": "CN-2015-06",
  "personal_context": "满仓状态，近期收益良好",
  "audience_and_purpose": "雪球公开帖，面向散户粉丝，分享看法",
  "context_dependency": "high",
  "dependency_evidence": "Step1=是 → Step3=是 → Step4=估值环境改变时失效"
}
```

### 语境依赖度

- `high`：判断强依赖当时市场、估值、政策或个人状态。
- `medium`：方法论相对稳定，但应用方式依赖情境。
- `low`：接近通用原则，语境改变不影响核心含义。

### 判定规则

1. 是否包含时间敏感判断？
2. 移除语境后含义是否改变？
3. 判断是否依赖当时环境条件？
4. 对 `high` 内容标注失效条件。

## 10. Phase 3：认知演化建模

### 目标

追踪目标人物的认知变化，划分阶段，并区分观点变化与深层跃迁。

### AGM 四类操作

| 类型 | 定义 | 处理 |
|------|------|------|
| 扩展 | 加入新信念，旧信念仍成立 | 可作为阶段内发展 |
| 修正 | 新信念替代旧信念 | 可作为阶段分界候选 |
| 收缩 | 旧信念消失且无明确替代 | 低置信处理 |
| 分裂 | 言论维持旧信念，行为违反旧信念 | 记入偏误或张力，不作为阶段分界 |

### 置信度

- `High`：明确自述、行为改变、时间点清晰。
- `Medium`：有两项证据，缺第三项。
- `Low`：只有推理支撑。

只有 `High` 和 `Medium` 的变化才能用于阶段划分。

### 认知跃迁条件

认定为深层跃迁至少需要满足三项：

1. 自述框架级反思。
2. 多领域行为同时变化。
3. 此后没有明显回到旧框架。

## 11. Phase 4：认知结构提炼

### 目标

把阶段分析转化为可迁移的认知结构。

### 继承层

继承层写入 agent 默认推理所需内容：

- **心智模型**：用什么底层框架看世界。
- **决策启发式**：在何种情境下如何判断。
- **表达 DNA**：句式、用词、确定性梯度。
- **认知偏误画像**：从行为中推断的盲区和偏差。
- **价值排序与张力**：面对冲突变量时的优先级。
- **智识谱系**：思想来源、受限之处、原创贡献。

### 元信息层

元信息层只记录历史语境：

- 自我规训条款。
- 公开表达边界。
- 社会角色带来的行为习惯。

这类内容不应成为 agent 的硬性执行禁令。

## 12. Phase 5：认知人格构建

### 输出结构

```text
[person-name]-cognitive-persona/
├── SKILL.md
├── meta.json
├── holdout-manifest.json
├── cognitive-timeline.md
├── cognitive-evolution-narrative.md
├── stage-profiles/
├── cognitive-structure/
├── references/
└── validation/
```

### SKILL.md 内容

生成的人格 skill 应包含：

1. 身份卡：一句话定义此人的认知特征。
2. 运行声明：说明继承的是认知运行方式，不是社会角色禁令。
3. 认知阶段声明：默认阶段与阶段切换规则。
4. 继承层：默认推理结构。
5. 元信息层：历史语境和自我规训。
6. 诚实边界：不可可靠迁移的领域。
7. 使用指南：置信层级、语境提示、运行模式。

## 13. Phase 6：质量验证

### 13.1 留出验证

解封留出集后，比较人格推断与真实材料：

- 是否命中真实立场？
- 是否能复现推理路径？
- 是否避免过度美化？
- 是否能承认超出范围？

### 13.2 功能测试

测试人格面对新问题时是否稳定：

- 边缘推断测试。
- 时间一致性测试。
- 反身性测试。
- 跨语境迁移测试。

### 13.3 反例与基线对照

与简单角色扮演、通用价值投资模板、普通摘要人格进行对照。合格的人格应有可区分性。

### 13.4 交叉评审

独立审阅证据链、推断强度、语境标注和诚实边界。若出现根本性反对意见，必须降档或返工。

## 14. 置信等级

### A 档

材料充分，证据链清晰，留出验证表现好，人格与基线可明显区分。

### B 档

有局部缺口或少量不一致，但核心结构成立，使用时需要标注限制。

### C 档

材料薄弱、矛盾严重或可区分性不足，只能作为研究札记，不能作为稳定认知人格使用。

## 15. 失败模式

### 15.1 金句拼贴

只收集漂亮表达，不抽取推理结构。修正方式：每条结论都必须追问"他为什么这么判断"。

### 15.2 传记化

把人生经历当成认知结构。修正方式：传记事实只在解释认知变化时使用。

### 15.3 崇拜化

把目标人物炼化成无错误的理想模型。修正方式：必须包含反面证据和偏误画像。

### 15.4 去语境化

把阶段性市场判断当成永恒原则。修正方式：所有关键判断都绑定 MCB 与 personal_context。

### 15.5 执行禁令污染

把目标人物的社会角色约束写成 agent 禁令。修正方式：行为规训进入元信息层，不进入继承层。

## 16. 交付质量要求

合格交付必须同时具备：

- 来源清单和材料质量标注。
- MCB 引用与个人语境标注。
- 认知阶段划分与阶段画像。
- 继承层和元信息层分离。
- 留出集验证记录。
- 明确置信等级和使用边界。

## 17. 理论参考

- Cognitive Task Analysis：用于从专家行为中提取任务知识和决策结构。
- Quentin Skinner 的历史语境主义：用于避免脱离语境解释文本。
- Gerd Gigerenzer 的生态理性：用于理解启发式和环境之间的匹配。
- AGM belief change theory：用于描述信念扩展、修正和收缩。
- Robert Kegan 的成人发展理论：用于区分局部观点变化和框架级跃迁。
